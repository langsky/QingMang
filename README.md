## 我为什么要做这个App
从2015年7月毕业到现在，我的日常工作都以解bug为主，主要对公司的Android手机项目中的Setting模块进行debug，因此我想尝试除去debug，去享受无到有开发一个App的乐趣。另一方面也是对我一年多的工作进行一个小总结。
## 我为什么选择轻芒
因为轻芒是个好东西。
## 我是怎么获取API的
我肯定获取不了轻芒的API，我使用了Jsoup对轻芒网页进行爬虫，然后解析出我需要的内容。不过Jsoup也有很多局限性，比如不能解析动态网页，所以为了显示文章，我只能跳转到文章原链接。
## 我为什么使用MVP
说实话，MVP，MVVM，我一直傻傻分不清，所以我姑且认为自己使用的MVP分层模式。
## 我为什么使用DataBinding
DataBinding是个好东西，在layout文件中稍微做下处理，就能节省大量的工作量，同时使用DataBinding便于解耦。
## 我为什么使用Glide
Glide是个好东西，使用它我就不必考虑复杂的异步加载图片的细节，以及各种缓存机制。当然，这些细节和机制我还是需要了解的。
## 我为什么使用ＲxJava
因为它处理异步任务十分简单。
## 我为什么使用ＧreenDao
因为它简单。
